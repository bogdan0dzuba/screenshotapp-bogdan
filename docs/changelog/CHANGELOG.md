# Changelog

## 2026-07-20 - 0.5.6

- `⌘⇧A` теперь сразу показывает встроенную рамку выбора области; внешний процесс захвата запускается только после отпускания мыши.
- На macOS 15.2+ выбранная область снимается напрямую через ScreenCaptureKit без запуска `/usr/sbin/screencapture`; для macOS 14 сохранен совместимый fallback.
- Импорт PNG выполняется в фоне, а размеры снимков читаются из метаданных ImageIO без полноразмерного декодирования. На текущей истории холодный проход по 20 снимкам сократился примерно с 1,85 с до 92 мс.
- Большой предпросмотр и уменьшенные миниатюры истории загружаются вне главного потока; миниатюры декодируются сразу в ограниченном размере.
- Декодирование превью сериализовано, отмененные запросы можно безопасно повторить, а длинные снимки получают больше детализации по мере увеличения при жестком лимите 16,8 млн пикселей.
- После получения PNG следующий хоткей доступен уже во время фонового импорта; смена папки блокируется до окончания сохранения, поэтому готовый снимок не может потеряться между каталогами.
- При двух быстрых снимках интерфейс сохраняет порядок самих захватов, а не скорость фонового копирования: поздно завершившийся старый импорт больше не переключает редактор поверх нового.
- Новый снимок без аннотаций больше не рендерится второй раз перед открытием редактора, а закрытые закрепленные окна сразу освобождают изображение из памяти.
- Добавлен отдельный RED/GREEN-контракт отзывчивости захвата; локальная, CI- и Universal-сборки повышены до 0.5.6 (20).

- Версия переходной локальной и Universal-сборки повышена до 0.5.3 (17).

- Версия локальной и Universal-сборки повышена до 0.5.2 (16).

## 2026-07-19 - 0.5.0

- Зафиксирован дизайн и план компактной полки с локальными метаданными: дата `19 июля, 14:50`, приложение/заголовок окна без новых разрешений, заметный счетчик, хоткей в заголовке, иконочные действия и более прозрачный Liquid Glass.
- Добавлены RED-проверки обратной совместимости метаданных, передачи источника через оба режима захвата и плотности новой полки.
- Добавлена Codable-модель источника, совместимое необязательное поле в элементах истории и JSON-проектах, формат подписи приложения/вкладки и заголовок даты без секунд.
- Обычный и прокручиваемый захват теперь запоминают активное приложение и доступный заголовок окна до скрытия полки, сохраняют их в проекте и восстанавливают после перезапуска без AppleScript или Accessibility.
- Временный источник прокручиваемого захвата очищается через `defer` и после успеха, и после ошибки.
- Провайдер источника напрямую использует `NSWorkspace.shared.frontmostApplication` и Core Graphics window list, сохраняя безопасный fallback на видимое чужое окно.
- Уточнен UI-контракт: служебный статус остается компактной строкой, а гибкий предпросмотр получает освобожденное место вместо фиксированных 250 pt.
- Развернутая полка стала плотнее: убрана очевидная подпись, счетчик и хоткей вынесены в заголовок, быстрые действия и нижний захват стали иконочными, история показывает дату/источник/размер, а Clear Liquid Glass и fallback получили примерно вдвое более прозрачную подложку.
- UI-контракт уточнен под семантический `Label(...).labelStyle(.iconOnly)`: подписи скрыты визуально, но остаются доступны VoiceOver и tooltip.
- Публичный release-контракт теперь требует описание локальных метаданных и их проверки в README, Privacy и GitHub Actions.
- Проверка метаданных подключена к локальной установленной сборке, Universal release и GitHub Actions; README и Privacy объясняют, что заголовок окна хранится только в локальном JSON-проекте.
- Добавлена RED-проверка helper локальной подписи после воспроизводимого падения упаковщика с SIGPIPE и кодом 141.
- Helper подписи теперь полностью дочитывает вывод Keychain: `security` больше не получает SIGPIPE от раннего завершения `awk`, и повторная локальная сборка не обрывается с кодом 141.
- Команда проверки локальной signing identity добавлена в README рядом с остальными воспроизводимыми проверками.
- Перед первой публикацией нормализованы окончания файлов, чтобы initial commit проходил `git diff --check` без whitespace-ошибок.
- Новый снимок больше не раскрывает намеренно свернутую полку; скрытая полка появляется после следующего снимка в компактном состоянии, а открытая остается открытой.
- Pinch-масштаб большого предпросмотра перенесен со внешнего `ScrollView`, который перехватывал trackpad-события, на полноразмерную поверхность изображения; прокрутка увеличенного кадра сохранена.
- Добавлен отдельный Universal release-сборщик: `arm64` и `x86_64` компилируются раздельно, объединяются через `lipo`, подписываются, упаковываются в стабильный ZIP и сопровождаются SHA-256.
- Добавлен GitHub Actions workflow на актуальном Intel macOS runner: обычные push/PR проверяют сборку, а тег `v*` публикует готовый Universal ZIP в GitHub Releases.
- README переработан в русскоязычную страницу продукта с прямой кнопкой скачивания, совместимостью, установкой, приватностью и безопасными GIF-демонстрациями без содержимого пользовательского экрана.
- В строку меню добавлена команда «Проверить обновления…», открывающая последний GitHub Release; фоновая самоустановка намеренно не включена до появления Developer ID и нотарификации.
- Версия локальной сборки повышена до 0.5.0 (14), добавлены проверки полноты release-упаковки и публичного репозитория.

- Исправлен повторный запрос «Запись экрана» после каждой сборки: приложение теперь подписывается постоянной локальной identity с устойчивым designated requirement, а не меняющимся ad-hoc `cdhash`.
- Сборка больше не удаляет рабочую установленную копию до проверки новой подписи; архив проверяется после распаковки, а уязвимая к Finder metadata прямая `.app`-копия в `Documents/dist` больше не создается.
- Локальная identity хранится в login Keychain и используется без добавления доверенного root/trust; это сохраняет устойчивую подпись и не расширяет системное доверие.

## 2026-07-19

- Отмена обычного захвата по `Esc` больше не показывает окно «Скриншот не готов»: завершение `screencapture` без созданного файла распознается как отмена, а реальные ненулевые ошибки остаются видимыми.
- Двузначный счетчик свернутой полки теперь всегда остается в одной строке: для значений от 10 используется компактный шрифт, фиксированный слот и запрет переноса, а внутренние отступы укладываются в ширину капсулы.
- Исправлено ложное раскрытие после переноса за стрелку: контрол больше не сравнивает координаты вокруг блокирующего `performDrag`, а обрабатывает `mouseDown`, `mouseDragged` и `mouseUp` отдельно. После первого превышения порога текущий жест уже не может стать кликом, даже если курсор вернулся назад.
- Стрелка сворачивания/разворачивания стала click/drag-контролом: короткое нажатие переключает состояние полки, а удержание с движением переносит окно и не раскрывает полку. Порог 4 pt защищает от случайного дрожания мыши.

## 2026-07-18

- Исправлено перемещение свернутой полки: drag-поверхность теперь занимает всю капсулу под содержимым, счетчик пропускает события мыши, а кнопка раскрытия остается отдельной активной зоной.
- Свернутая капсула уменьшена с 64 × 36 до 60 × 34 pt, стрелка и счетчик увеличены; почти непрозрачная черная подложка заменена легким системным материалом с более светлым стеклянным бликом.
- Настройка хоткея больше не показывает непонятный ряд символов и переключателей: модификаторы стали обычными macOS-флажками с подписями Command, Shift, Option и Control, буква получила видимую подпись, а текущая комбинация расшифровывается как `Command (⌘) + Shift (⇧) + A`.
- Компактное отображение сочетания приведено к ожидаемому порядку `⌘⇧A` вместо `⇧⌘A`.
- В заголовок полки добавлена шестеренка, открывающая настройки. Новый сохраняемый режим «Закрывать редактор после копирования» включен по умолчанию и закрывает текущее окно после успешных `Ctrl+C`, `⌘C` или кнопки «Копировать»; при ошибке окно остается открытым.
- Кнопка сворачивания/разворачивания получила цельную активную область 28 × 28 pt вокруг стрелки; первый клик больше не требует точного попадания в пиксели символа.
- Исправлено копирование после захвата: активный редактор принимает `⌘C` и `Ctrl+C`, большой предпросмотр в полке копируется обычным кликом, а стандартная кнопка редактора использует `⌘C` вместо `⌘⇧C`.
- У инструментов редактора увеличена активная зона до всего квадрата 36 × 32 pt; клик рядом с символом внутри квадрата теперь выбирает инструмент.
- В большую область предпросмотра развернутой полки добавлен pinch на трекпаде: снимок увеличивается, после увеличения прокручивается двумя пальцами, а предел масштаба автоматически повышается для очень длинных страниц. Такой же жест работает на холсте редактора в диапазоне 25–800%.
- Исправлен перенос снимков из полки: перетаскивание окна ограничено отдельными drag-зонами, а каждый снимок передает реальный PNG и file URL вместо движения всей панели.
- Копирование выбранного снимка принимает `⌘C` и `Ctrl+C`, в том числе когда событие пришло в видимую non-activating полку до получения ею key-status; вставка в другие macOS-приложения выполняется стандартным `⌘V`.
- Размер редактора теперь рассчитывается из размеров снимка и доступного места текущего дисплея; маленькие снимки открываются в компактном окне, большие ограничиваются экраном, панели инструментов остаются доступны через горизонтальную прокрутку.
- Основная подписанная кнопка полки теперь запускает выбор области для снимка «С прокруткой»; автоматическая склейка принимает прокрутку как вниз, так и вверх, а обычная область остается на хоткее и отдельной иконке.
- Исправлен первый клик по кнопке сворачивания/разворачивания: полка стала non-activating panel, принимает `acceptsFirstMouse` через специализированный `NSHostingView`, а SwiftUI-кнопка использует `allowsWindowActivationEvents`; первый клик больше не расходуется только на активацию приложения.
- После любого обычного или прокручиваемого снимка редактор открывается автоматически, а сам снимок продолжает накапливаться в полке истории.
- Прокручиваемый захват стал непрерывным: пользователь просто листает страницу, одинаковые кадры пропускаются, кадры с перекрытием добавляются автоматически, для управления доступны пауза, продолжение, отмена кадра и завершение.
- Длинные снимки в редакторе теперь подгоняются по ширине и прокручиваются по вертикали вместо уменьшения всей страницы до нечитаемого превью.
- Панель прокручиваемого захвата исключена из снимка через системный window sharing и появляется на том дисплее, где пользователь выделял область.
- Добавлена защита от слишком резкой прокрутки без перекрытия и безопасный лимит 80 исходных кадров.
- Сборщик автоматически выбирает совместимый локальный macOS SDK при несовпадении Swift compiler и системного SDK; нативный Liquid Glass компилируется при доступном SwiftUI 7, на старом SDK используется существующий material fallback.
