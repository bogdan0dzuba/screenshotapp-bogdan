# Понятные имена файлов и хранение без лимита

## Цель

Новые снимки должны иметь понятные локальные имена, совпадающие по смыслу с историей приложения. Пользователь также должен иметь возможность полностью отключить автоматическое удаление старых снимков.

## Имена новых снимков

- Базовый формат: `21 июля, 10.32 - Telegram`.
- В имени используется только название приложения из `CaptureSource.applicationName`. Заголовок окна, вкладка и адрес сайта не добавляются.
- Если приложение определить не удалось, используется только дата и время: `21 июля, 10.32`.
- Символы, недопустимые или неудобные в имени файла, заменяются пробелом; повторяющиеся пробелы и разделители нормализуются.
- При совпадении имен выбирается первый свободный вариант: `… - Telegram (2)`, `… - Telegram (3)`.
- Итоговый PNG, исходный PNG и проект редактора используют один базовый stem: `.png`, `.source.png`, `.project.json`.
- Существующие снимки не переименовываются. Старый и новый форматы распознаются одновременно, чтобы история, удаление и редактор продолжили работать.

## Хранение

- В разделе «История -> Хранение» появляется переключатель `Автоматически удалять старые снимки`.
- Для существующих установок переключатель включен по умолчанию, поэтому текущее поведение не меняется без выбора пользователя.
- Когда переключатель включен, продолжают работать лимит количества и максимальный возраст.
- Когда переключатель выключен, приложение не удаляет снимки автоматически ни по количеству, ни по возрасту. Ручные команды «Удалить» и «Очистить историю» продолжают работать.
- Настройка сохраняется в `UserDefaults` и применяется после нажатия существующей кнопки применения настроек хранения.
- Элементы управления количеством и возрастом отключаются визуально, когда автоматическое удаление выключено, но их значения сохраняются для последующего включения.

## Безопасность данных

- Автоматическая очистка работает только с файлами, которые распознаны как собственные файлы ScreenshotApp.
- Создание трех связанных файлов остается транзакционным: при ошибке частично созданные файлы удаляются.
- Выключение автоудаления никогда не запускает очистку уже накопленных файлов.

## Проверка

- RED/GREEN-тест форматирования имени с приложением, без приложения, очистки символов и разрешения коллизий.
- Тест совместимости классификатора со старым и новым форматами.
- Тест retention-политики: включенный режим ограничивает историю, выключенный сохраняет все элементы независимо от количества и возраста.
- UI-контракт проверяет переключатель, сохранение настройки и отключение лимитов.
- Полная сборка, установка версии 0.5.8 и Universal-релиз для `arm64 + x86_64`.
